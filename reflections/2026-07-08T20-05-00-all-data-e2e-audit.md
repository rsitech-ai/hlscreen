## Reflection
- **Failure modes observed:** The live path worked, but the TUI confidence detail label used `gap:N` for incomplete feature-window count, which could be misread as reconnect/data gaps.
- **Root cause:** `format_confidence_counters` read `row.confidence.incomplete_windows.len()` and rendered it as `gap`, while reconnect/data gaps are tracked separately through `ConfidenceReason::ReconnectGap` and recorded `data_gaps`.
- **Fix that resolved it:** Renamed the counter to `window:N` and updated golden tests plus generated screenshots.
- **What improved score/quality:** Fresh live proof was paired with replay parity, real-data screen commands, negative probes, official-doc review, and screenshot inspection instead of treating a single passing test as enough.
- **Useful command-level evidence:** 180s all-symbol run `allpairs-e2e-20260708-195413`; post-fix 60s all-symbol run `allpairs-postfix-20260708-195842`; replay parity baseline then pass; full Rust/release gates green.
- **Branch comparison insight (if multiple attempts):** The post-fix run proved the changed TUI label on the current binary without changing live WebSocket behavior.

## Reusable Lesson
- **Pattern that worked:** Treat live smoke, replay parity, and UI semantic review as one audit bundle for trading-adjacent tools.
- **Pattern to avoid:** Do not reuse compact operator labels when the backing field means something different.
- **Where to apply next:** Any future TUI metric rename, confidence reason addition, or top-of-book proxy expansion.

- **Final chosen approach:** Fresh bounded all-symbol public smoke plus source/validation audit.
- **Commit/rollback decision:** Commit after report/notes; rollback is safe because only audit artifacts, screenshots, and a TUI label were changed.
